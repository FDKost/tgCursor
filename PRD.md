# Product Requirements Document: Telegram Task Tracker Bot

## Executive Summary

**Telegram Task Tracker Bot** — это Telegram-бот для управления личными задачами: быстрый инбокс, статусы, дедлайны, напоминания и ежедневный обзор прогресса прямо в чате.

MVP фокусируется на **скорости ввода** (1 команда → 1 задача), **контроле дедлайнов** (today/overdue) и **inline-управлении** задачами без «лишних экранов».

## Mission

**Миссия продукта:** помочь пользователю не терять задачи и доводить их до результата за счёт максимально простого ввода и системных напоминаний.

### Основные принципы
1. **Fast capture** — задача добавляется быстрее, чем её можно забыть.
2. **Focus** — интерфейс подсказывает «что делать сейчас» (doing/today/overdue).
3. **Low friction** — управление задачами через inline‑кнопки, минимум команд.
4. **Respect user time** — напоминания полезные, настраиваемые, без спама.
5. **Private by default** — безопасное хранение данных и минимум логирования контента.

## Target Users

### Основная целевая аудитория
- **Соло‑специалисты** (разработчики, фрилансеры): много мелких задач, нужен быстрый инбокс.
- **Менеджеры/лиды**: ежедневный список, контроль просрочек, фокус на «горящем».
- **Студенты**: дедлайны и регулярные напоминания.

### Ключевые потребности пользователей
1. Зафиксировать задачу в 1 шаг.
2. Вести задачи по статусам (todo/doing/done).
3. Видеть «на сегодня» и просроченное.
4. Получать напоминания и быстро реагировать (закрыть/перенести).
5. Находить задачу по тексту/тегу.

## Goals & Success Metrics

### Product Goals
- Упростить добавление задач и снизить «утечки» (забытые задачи).
- Дать ежедневный контроль: today/overdue/doing.
- Повысить доведение до done за счёт напоминаний и быстрых действий.

### Success Metrics (примеры)
- **Activation**: ≥1 созданная задача в первые 5 минут после `/start`.
- **Retention D7**: доля активных пользователей на 7-й день.
- **Task completion rate**: доля задач, переведённых в `done`.
- **Reminder effectiveness**: доля напоминаний, после которых задача закрыта/перенесена ≤ 24ч.

## MVP Scope

### ✅ In Scope — Core Functionality

**Задачи:**
- Создание задачи: `/add <текст>` (минимум: title).
- Статусы: `todo`, `doing`, `done` (опционально: `inbox`, `archived`).
- Приоритет: `low / normal / high`.
- Дедлайн: установка/снятие даты/времени.
- Теги: добавить/удалить (минимум: список тегов на задаче).

**Списки и поиск:**
- `/list` (по умолчанию: today + overdue + doing).
- Быстрые списки: `/todo`, `/doing`, `/done` (и/или кнопки).
- Поиск: `/search <запрос>` (минимум по title; опционально по тегам).
- Пагинация списка («показать ещё») при длинных выдачах.

**Напоминания:**
- Напоминание до дедлайна (по настройке пользователя).
- «Тихие часы» или временное окно отправки напоминаний.
- Inline‑действия в напоминании: Done / перенести / открыть задачу.

**Экспорт:**
- Экспорт задач в чат (текст) и/или `.csv`.

### ✅ UX / Telegram Interface

**Entry points:**
- `/start`: приветствие + 2–3 подсказки + пример команды `/add`.
- (Опционально) свободный текст: предложить создать задачу из сообщения.

**Основные команды (предложение):**
- `/add <текст>` — создать задачу
- `/list` — обзор
- `/todo` `/doing` `/done` — быстрые списки
- `/task <id>` — карточка задачи
- `/search <запрос>` — поиск
- `/settings` — часовой пояс и напоминания
- `/help` — справка

**Карточка задачи (сообщение + inline‑кнопки):**
- Поля: title, status, priority, due_at, tags, created/updated, (опц.) description
- Inline‑кнопки MVP:
  - ✅ Done
  - ▶️ Doing / 📝 Todo
  - ⏰ Set deadline (сценарии: сегодня/завтра/ввести)
  - ➕1d (если есть due_at)
  - 🏷 Tags
  - 🗑 Delete (с подтверждением) или Archive

### ❌ Out of Scope (not MVP)
- Командная работа (общие списки/назначение/права).
- Сложные повторения (RRULE), зависимости задач, диаграммы.
- Интеграции (Notion/Jira/Trello/Calendar) — бэклог.
- Полноценный NLP‑парсинг дат («в понедельник 18:00») — позже.

## Functional Requirements

### Tasks
- FR-1: Создать задачу с минимум полями: `id`, `telegram_user_id`, `title`, `status`, `created_at`.
- FR-2: Изменять статус задачи.
- FR-3: Устанавливать/снимать дедлайн.
- FR-4: Устанавливать приоритет.
- FR-5: Добавлять/удалять теги.
- FR-6: Выдавать списки задач с фильтрами (status, today, overdue).

### Reminders
- FR-7: Планировать и отправлять напоминания согласно настройкам.
- FR-8: Не отправлять напоминания в «тихие часы» (или соблюдать временное окно).
- FR-9: Поддерживать действия из напоминания: done / snooze (перенос) / open task.

### Settings
- FR-10: Хранить часовой пояс пользователя.
- FR-11: Хранить и применять предпочтения по напоминаниям (например, за N минут/часов; утром в день дедлайна).

### Export
- FR-12: Экспортировать задачи (text/CSV).

## Non-Functional Requirements

- NFR-1: **Secrets** — токен и секреты только через env/.env (не в репозитории).
- NFR-2: **Error handling** — ошибки Telegram API не показывают пользователю трейсбеки; корректные сообщения.
- NFR-3: **Performance** — `/list` и карточка отвечают примерно за 1–2 секунды при нормальной нагрузке.
- NFR-4: **Scalability** — хранение данных допускает рост до десятков тысяч задач.
- NFR-5: **Timezone correctness** — корректная работа с часовыми поясами.
- NFR-6: **Logging** — логирование событий без хранения приватного текста (или с минимизацией).

## Data Model (Draft)

### Entities
- **User**
  - `telegram_user_id` (unique)
  - `timezone`
  - `reminder_prefs`
  - `created_at`, `updated_at`
- **Task**
  - `id` (int/uuid)
  - `telegram_user_id`
  - `title`
  - `description` (optional)
  - `status` (enum)
  - `priority` (enum)
  - `due_at` (datetime, optional)
  - `tags` (array / m2m)
  - `created_at`, `updated_at`, `completed_at` (optional)
- **Reminder**
  - `id`
  - `task_id`
  - `scheduled_at`
  - `sent_at` (optional)
  - `state` (scheduled/sent/cancelled)

### Business Rules
- Задача в `done` не должна напоминаться (все будущие напоминания отменяются).
- «Просрочено» = `due_at < now` и статус не `done` (и не `archived`, если есть).

## Analytics / Events

События (без приватного текста):
- `user_started`
- `task_created`
- `task_completed`
- `task_deadline_set`
- `reminder_sent`
- `reminder_action_done`
- `reminder_action_snooze`

## Risks & Edge Cases

- Не настроен часовой пояс → некорректные дедлайны/напоминания.
- Флуд задачами → rate limit на пользователя.
- Очень длинные списки → пагинация/«показать ещё».
- Удаление по ошибке → подтверждение и/или Archive вместо Delete.

## Acceptance Criteria (MVP)

- Пользователь может:
  - создать задачу через `/add`
  - увидеть обзор `/list` с today/overdue
  - открыть `/task <id>` и переключить статус (включая done) inline‑кнопкой
  - поставить дедлайн и получить напоминание
  - отфильтровать задачи по статусам
  - экспортировать задачи (text/CSV)
- В репозитории нет секретов; всё через env/.env.
- Ошибки Telegram API не приводят к «падению» в диалоге; пользователю показывается понятное сообщение.

## Roadmap (Post‑MVP Backlog)

- Проекты/списки (work/personal) и быстрые переключатели.
- Повторяющиеся задачи.
- Совместные списки (шаринг).
- Интеграции: Google Calendar, Notion, Jira.
- Умный парсинг естественного языка для дедлайнов.
